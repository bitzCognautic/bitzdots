# Waybar Modules

14 custom modules are configured in `waybar/config.jsonc` with performance-optimized polling intervals.

## Module Reference

| Module | Custom Script | Interval | Purpose |
|--------|---------------|----------|---------|
| `custom/workspaces` | — | 3s | Active workspace names (uses `hyprctl workspaces -j`) |
| `custom/cpu` | — | 5s | CPU usage percentage |
| `custom/memory` | — | 10s | RAM usage |
| `custom/network` | — | 10s | Network status and SSID |
| `custom/recording` | `scripts/recording-indicator.sh` | 10s | Blinking `` icon when `wf-recorder` is active |
| `custom/power-profiles` | `scripts/power-profile.sh` | 30s | Current power profile (performance/balanced/power-saver) |
| `custom/power-switch` | `scripts/power-profile-switch.sh` | on-click | Toggle power profile on click |
| `custom/notification` | — | 10s | Notification count from SwayNC |
| `custom/clock` | — | 30s | Date and time display |
| `custom/brightness` | — | 5s | Screen brightness level (with `timeout 2`) |
| `custom/media` | `scripts/media.sh` | on-change | Now playing media (playerctl) |
| `custom/tray` | — | — | System tray icons |
| `custom/wireplumber` | — | on-change | Audio sink indicator |

## Performance Optimizations

- **No `"interval": "once"`** — Waybar 0.15.0 uses `"interval": 0` instead
- **Workspaces use `hyprctl workspaces -j`** instead of heavier `hyprctl clients -j`
- **All intervals ≤ 30 seconds** for low CPU impact
- **Minimal CSS transitions** — no hover effects on notifications
- **Recording indicator** polls `pgrep` rather than watching for file changes

## Custom Styling

The `waybar/style.css` uses colors generated by wallust, so the bar automatically matches your wallpaper theme. Key style properties:

- **Background**: Transparent with blur
- **Font**: `JetBrains Mono` (Nerd Font)
- **Border**: Bottom border with accent color
- **Padding**: Comfortable spacing between modules
- **Tooltips**: Disabled for performance

## Adding a Module

1. Add the module to `waybar/config.jsonc` in the desired position
2. Add corresponding CSS classes in `waybar/style.css`
3. For custom scripts, place them in `scripts/` and make them executable
4. Reload waybar with `SUPER+SHIFT+C`
