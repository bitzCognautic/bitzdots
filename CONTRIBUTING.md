# Contributing

Thanks for wanting to contribute to bitzdots!

## How to Contribute

1. **Fork** the repo
2. **Create a branch** (`git checkout -b feature/my-change`)
3. **Make your changes**
4. **Test** them on your own setup
5. **Commit** (`git commit -m "description of change"`)
6. **Push** and open a **Pull Request**

## Guidelines

- Keep the **low-end focus** — every addition should be optional or lightweight
- **Document your changes** — update relevant docs if adding features
- **Follow the style** — match existing code patterns and conventions
- **No bloat** — if a feature adds significant overhead, make it opt-in

## Reporting Issues

Use the bug report template. Include your distro, Hyprland version, and relevant logs.

## Feature Requests

Open an issue with the feature request template. Explain the use case and why it fits bitzdots.

## Code of Conduct

Be respectful. This is a hobby project — keep it fun.
